# Sprint 3 — Outfit Library

## Objective

Create a practical outfit guide using only garments currently owned.

This is not a style inspiration document. It is an operational guide for daily dressing.

## Files

### Modify

- `docs/05_Outfit_Library.md`

### Review

- `docs/04_Wardrobe.md`
- `docs/03*` (all garment category specs)
- `docs/02*` (Decision Framework)

## Outfit Categories

Generate outfits for:

- Office
- Business Casual
- Casual Friday
- Weekend
- Date Night
- Wedding
- Festivals
- Travel

If a category cannot be completed using the existing wardrobe, explicitly state why rather than inventing missing garments.

## Outfit Structure

Each outfit should include:

- Trousers
- Shoes
- Top
- Layering
- Accessories (optional)

Follow the outfit construction hierarchy already defined in the repository.

## Explanation

For every outfit, explain:

- Why it works
- Why it suits the user's proportions
- Why it suits Deep Autumn colouring
- How it balances masculinity and elegance

Keep explanations concise.

## Additional Section

Create a "Fallback Outfits" section with simple combinations suitable when no planning is required.

## Success Criteria

The document should allow the user to dress appropriately for common occasions without needing to think through combinations from scratch.

## Deliverables

- Completed `docs/05_Outfit_Library.md`
