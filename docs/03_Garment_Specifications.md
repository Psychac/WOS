# 03 – Garment Specifications

## Purpose

This document serves as the index and navigation guide for evaluating well-designed garments.

## Document Structure

Each split document follows this consistent structure:

1. **Purpose** — One-sentence statement
2. **Part I – Garment Fundamentals** — Cross-referenced core concepts
3. **Category Philosophy** — Design philosophy specific to this category
4. **Universal Principles** — Rules that apply to all items in the category
5. **Foundation vs Accent** — How items are classified by usage
6. **Common Pitfalls** — Errors to avoid across the category
7. **Individual Garment Specifications** — For each item, organized as:
   - Importance Tier
   - Quick Spec (summary table)
   - Role
   - When I Need It
   - Design Specification (Fit, Fabric, Colours, Pattern, Construction, etc.)
   - Outfit Compatibility
   - Wardrobe Notes
   - Avoid (things to reject)
   - Evaluation Checklist (weighted attributes)
   - Verdict (Buy / Consider / Skip)

## Editing Notes

When making changes to garment specifications:

1. **Edit the appropriate split document** (e.g., 03A for shirt changes)
2. **Update Layer 1 documentation** first (the human knowledge base)
3. **Verify consistency** across related garments (e.g., matching Colour recommendations)
4. **Update evaluation checklists** if priorities change
5. **Do not edit this index directly** — it reflects the structure, not individual specifications

---

## Architecture Rationale

This modularization was chosen to:

1. **Reduce complexity** — From 9,000+ lines to ~1,500 lines per document
2. **Enable independent refactoring** — Each category can be improved separately in Phase 3
3. **Preserve self-containedness** — Part I fundamentals are included in every document
4. **Maintain consistency** — Universal principles are stated in each category
5. **Support independent commits** — Each split can be reviewed and merged independently
