# Documentation Style Guide

## Philosophy

The repository is a technical specification.

Write like engineering documentation rather than a fashion blog.

---

# Tone

Be:

- Objective
- Concise
- Precise
- Decision-oriented

Avoid:

- Marketing language
- Subjective praise
- Trend language

---

# Document Structure

Prefer:

Role

↓

When I Need It

↓

Design Specification

↓

Outfit Compatibility

↓

Wardrobe Notes

↓

Evaluation Checklist

---

# Evaluation Philosophy

Evaluation order should reflect the garment.

Always begin with the garment's defining characteristic.

Examples:

Shirts

→ Collar

Jeans

→ Wash

Chinos

→ Rise

Outerwear

→ Silhouette

Footwear

→ Last

Tailoring

→ Silhouette

Never force identical evaluation templates.

---

# Decision Rationale

Explain:

Why is this option preferred?

Why is it better than alternatives?

What trade-offs are being accepted?

Avoid simply describing features.

---

# Writing Principles

Prefer:

Short paragraphs.

Consistent terminology.

Minimal duplication.

Reusable explanations.

---

# Scope

Every specification should answer:

"What should the ideal version of this garment look like?"

Avoid:

- History
- Encyclopedic information
- Irrelevant styling advice
- Excessive theory

---

# Refactoring

When editing:

1. Preserve intent.
2. Reduce duplication.
3. Improve consistency.
4. Simplify wording.
5. Do not expand unnecessarily.

---

# Repository Rule

Optimize for retrieval.

Not document length.

Small, coherent documents are preferred over large monolithic documents.
